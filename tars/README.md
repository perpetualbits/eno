# tars/ — development snapshots

Point-in-time tarballs of subsystem work. These are historical checkpoints,
not release artifacts. They capture state at the end of a session so an
earlier working baseline can be recovered without `git` archaeology.

All are purely historical. The canonical source is the live repo.
